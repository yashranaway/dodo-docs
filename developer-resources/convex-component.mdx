## Installation

<Steps>
<Step title="Install the package">
Run the following command in your project root:

```bash
npm install @dodopayments/convex
```
</Step>
<Step title="Add Component to Convex Config">
Add the Dodo Payments component to your Convex configuration:

```typescript
// convex/convex.config.ts
import { defineApp } from "convex/server";
import dodopayments from "@dodopayments/convex/convex.config";

const app = defineApp();
app.use(dodopayments);
export default app;
```
</Step>
<Step title="Set up environment variables">
Set up environment variables in your Convex dashboard (**Settings** → **Environment Variables**) or `.env` (if using local development):

```env
DODO_PAYMENTS_API_KEY=your-api-key
DODO_PAYMENTS_ENVIRONMENT=test_mode
DODO_PAYMENTS_WEBHOOK_SECRET=your-webhook-secret
```

<Warning>
Never commit your secrets to version control. Always use Convex environment variables for sensitive information.
</Warning>
</Step>
</Steps>

## Checkout Functions

<Info>
Dodo Payments supports three types of payment flows for integrating payments into your website. This component supports all types of payment flows.
</Info>

- **Checkout Sessions:** Create secure, customizable checkout experiences with pre-configured product carts and customer details. (Recommended)
- **Static Payment Links:** Instantly shareable URLs for quick, no-code payment collection.
- **Dynamic Payment Links:** Programmatically generate payment links with custom details using the API.

<AccordionGroup>
<Accordion title="Static Checkout">

### Usage
```typescript
const result = await staticCheckout(ctx, {
  productId: "prod_123",
  quantity: "1",
  returnUrl: "https://example.com/success"
});
```

### Supported Parameters
<ParamField query="productId" type="string" required>
Product identifier (e.g., <code>prod_123</code>).
</ParamField>
<ParamField query="quantity" type="string">
Quantity of the product.
</ParamField>
<ParamField query="returnUrl" type="string">
URL to redirect after successful checkout.
</ParamField>
<ParamField query="fullName" type="string">
Customer's full name.
</ParamField>
<ParamField query="firstName" type="string">
Customer's first name.
</ParamField>
<ParamField query="lastName" type="string">
Customer's last name.
</ParamField>
<ParamField query="email" type="string">
Customer's email address.
</ParamField>
<ParamField query="country" type="string">
Customer's country.
</ParamField>
<ParamField query="paymentCurrency" type="string">
Specify the payment currency (e.g., <code>USD</code>).
</ParamField>

### Response Format
Static checkout returns a JSON response with the checkout URL:
```json
{
  "checkout_url": "https://checkout.dodopayments.com/..."
}
```
</Accordion>

<Accordion title="Session Checkout (Recommended)">

### Usage
```typescript
const result = await sessionCheckout(ctx, {
  payload: {
    product_cart: [{ product_id: "prod_123", quantity: 1 }],
    customer: { email: "user@example.com" },
    return_url: "https://example.com/success"
  }
});
```

### Supported Payload Fields
<ParamField body="product_cart" type="array" required>
Array of products with <code>product_id</code> and <code>quantity</code>.
</ParamField>
<ParamField body="customer" type="object">
Customer information including <code>email</code>, <code>name</code>, and <code>phone_number</code>.
</ParamField>
<ParamField body="billing_address" type="object">
Billing address with <code>street</code>, <code>city</code>, <code>state</code>, <code>country</code>, and <code>zipcode</code>.
</ParamField>
<ParamField body="return_url" type="string">
URL to redirect after successful checkout.
</ParamField>
<ParamField body="billing_currency" type="string">
Specify the payment currency (e.g., <code>USD</code>).
</ParamField>
<ParamField body="feature_flags" type="object">
Control checkout features like <code>allow_discount_code</code>, <code>allow_currency_selection</code>.
</ParamField>
<ParamField body="customization" type="object">
Customize checkout appearance with <code>theme</code> and display options.
</ParamField>
<ParamField body="metadata" type="object">
Custom metadata to attach to the payment.
</ParamField>

### Response Format
Session checkout returns a JSON response with the checkout URL:
```json
{
  "checkout_url": "https://checkout.dodopayments.com/session/..."
}
```
</Accordion>

## Customer Portal Function

The Customer Portal Function enables you to seamlessly integrate the Dodo Payments customer portal into your Convex application.

### Usage
```typescript
const result = await customerPortal(ctx, {
  send_email: false
});
```

### Parameters
<ParamField query="send_email" type="boolean">
If set to <code>true</code>, sends an email to the customer with the portal link.
</ParamField>

<Info>
The customer portal function automatically uses the authenticated user's `customerId` from the `identify` function you provide in the configuration.
</Info>

<Warning>
Returns an error if the user is not authenticated or if the `identify` function returns null.
</Warning>

## Webhook Handler

- **Method:** Only POST requests are supported. Other methods return 405.
- **Signature Verification:** Verifies the webhook signature using <code>DODO_PAYMENTS_WEBHOOK_SECRET</code>. Returns 401 if verification fails.
- **Payload Validation:** Validated with Zod. Returns 400 for invalid payloads.
- **Error Handling:**
  - 401: Invalid signature
  - 400: Invalid payload
  - 500: Internal error during verification
- **Event Routing:** Calls the appropriate event handler based on the payload type.

## Frontend Usage

<Info>
Use the checkout functions from your React components with Convex hooks.
</Info>

<CodeGroup>
```tsx React Checkout Component expandable
import { useAction } from "convex/react";
import { api } from "../convex/_generated/api";

export function CheckoutButton() {
  const createSessionCheckout = useAction(api.payments.createSessionCheckout);

  const handleCheckout = async () => {

}
```
